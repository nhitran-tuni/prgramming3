Detailed Description

Runner asks players' for actions and executes them. (The actions, not the players :D.)

Usage:

  1. Set control objects for the players.
  2. Call run(), which starts communicating with control objects to ask for actions which are then executed. For each action executed, actionPerformed-signal is emitted.
  3. If run() returns a pointer to a control object, the player managed by the object is in turn but the object couldn't give the next move. Fix the state of the object or exchange it to a new one, and return to step 2.
  4. If run() returns a nullptr, game has ended.

If class method is not nothrow, 'std::bad_alloc' can always leak. (out of memory)
